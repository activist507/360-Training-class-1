Follow the instructions in each section below. Write all your CSS in the <style> tag at the top of this file. Do not change any HTML unless a task tells you to. Each section tells you exactly what to target and what effect to produce — the visual result is up to you, as long as the requirement is met.

Part 1 — Selectors

Complete each task using the exact selector type named in brackets. Don't use a class where the task calls for an element, ID, or attribute selector, etc.

  1. [Element] Give every <p> in this section a different text color than the default.
  2. [Class] Style .highlight-card with a background color, padding, and a border-radius.
  3. [ID] Style #feature-box with a distinct border.
  4. [Descendant] Style only the <a> tags that are inside nav.site-nav (not links elsewhere on the page).
  5. [Pseudo-class: hover] Make links in .link-list change color when hovered.
  6. [Pseudo-class: nth-child] In .zebra-list, give every even <li> a different background color than the odd ones.
  7. [Attribute] Style every input[required] in the form with a colored left border.
  8. [Attribute] Style input[type="email"] differently from other inputs in the form.
  9. [Adjacent sibling] Style the <p> that comes immediately after h3.sibling-target — no other paragraph in that section should be affected.
  10. [General sibling] Style every <p> that comes after h3.sibling-target (not just the first one) using ~ — compare the visual result to Task 9.
  11. [Universal] Add box-sizing: border-box; to every element on the page using *.
  12. [Pseudo-element: ::before / ::after] Add decorative content before or after each .tag element (e.g. a symbol or bracket) without editing the HTML.
  13. [Pseudo-element: ::first-letter] Give the first letter of .drop-cap a noticeably larger font size.
  14. [Pseudo-element: ::selection] Change the highlight color used when a user selects/drags over text on the page.
  15. [Grouping] Write one rule that applies the same heading style to h1, h2, h3 at once.
  16. [Chaining] Style p.intro (a paragraph that also has the class "intro") differently from other paragraphs.

Part 3 — Typography

Complete each task below.

  1. [font-family] Give .type-family a font stack with at least two fallback fonts.
  2. [font-size + clamp] Give .type-fluid a font-size using clamp(min, preferred, max) so it scales with the viewport.
  3. [font-weight] Give .type-weight a numeric font-weight (not "bold").
  4. [font-style] Make .type-style italic.
  5. [line-height] Set a comfortable unitless line-height (around 1.5) on .type-body.
  6. [line-height] Set a tighter unitless line-height (around 1.1–1.2) on .type-heading.
  7. [letter-spacing] Give .type-open slightly increased letter-spacing using em.
  8. [letter-spacing] Give .type-tight slightly negative letter-spacing using em.
  9. [word-spacing] Increase the word-spacing on .type-wordspace.
  10. [text-align] Center .type-center.
  11. [text-indent] Indent the first line of .type-indent.
  12. [max-width / ch] Constrain .type-readable to a readable line length using ch units.
  13. [text-decoration] Give .type-decoration a non-default text-decoration (e.g. a dotted underline).
  14. [text-transform] Make .type-caps render in uppercase without changing the HTML text.
